# Проект из 3 этапов:
Создан сервис оценки недвижимости. От построения пайплайна и оптимизации модели до вывода в продакшн и настройки мониторинга. FastAPI, Docker Compose, Prometheus, Grafana, Airflow, DVC, MLflow, Optuna, PostgreSQL

Краткое описание проекта : Прогнозирования стоимости недвижимости 
Цель — улучшить ключевые метрики модели для предсказания стоимости квартир Яндекс Недвижимости. 
Использованные технологии:
Visual Studio Code,
Jupyter Notebook,
база данных PostgreSQL с данными для обучения,
база данных PostgreSQL с данными для MLflow,
объектное хранилище для MLflow,
сервисы MLflow (Tracking Server и Model Registry),
Dиртуальная машина для проведения всех этапов,
Библиотеки Pipeline, ColumnTransformer, autofeat, seaborn, matplotlib, catboost, SequentialFeatureSelector, TPESampler, KMeans, Optuna

Этап 2: Проведение EDA
Ключевые шаги EDA:
1. Знакомство с данными, изучение типов данных, проверка наличия пропущенных значений
2. Визуализация данных: графики и диаграммы для визуализации распределений признаков и возможных взаимосвязей между ними.
3. Статистический анализ: Применение статистических методов для изучения центральных тенденций, разброса и корреляций между признаками.
4. Предварительные выводы: На основе проведённого анализа сформулировал предварительные выводы о данных, которые помогут в дальнейшем этапе моделирования.


Я бы сформулировал следующие выводы по результатам EDA:
1. Наиболее широко встречаются цены на квартиры из диапазона от 6 млн. до 15 млн. Средняя самая частая цена находится в области 8 млн.
2. Диапазон средней цены на кваритру повышается при увеличении комнат в квартире.
3. Этаж, наличие лифта, аппартаменты, высота потолков оказывают незначительное влияние на диапазон цены
4. Тип здания заметно смещает диапазон цены за квартиру
5. Год постройки здания имеет влияние на цену. Новые здания коррелируют с изменением стоимость но плавно с течением времени
6. Таким образом набор признаков, которые представляются важными:
    - rooms
    - building_type_int
    - build_year
    - price
7. Признаки, которые можно попробовать сгенерировать дополнительно:
    - средняя площадь квартиры в зависимости от количества комнат
    - возраст здания на текущий год
    - общее количество квартир по комнатам на этаже
    - общее количество квартир по этажам
    - регион, используя кластеризацию по географическим координатам
8. Признаки, которые можно исключить из рассмотрения:
    - studio - так как там значения все False, что не дает никакой информации для модели
    - longitude и latitude - так как они по отдельности могут плохо пониматься моделью

Этап 3: Генерация признаков и обучение модели
На данном этапе были сформированы следующие признаки:
- средняя площадь квартиры в зависимости от количества комнат
- возраст здания на текущий год
- общее количество квартир по комнатам на этаже
- общее количество квартир по этажам
- регион, используя кластеризацию по географическим координатам
- признаки сгенерированные автоматически "autofeat" с трансформаторами ["^2", "1+"]
Созданный пайплайн по признакам на основе моих выводов показал плохой результат, хуже чем на сырой базе
Модель для обучения использовалась : CatBoostRegressor
